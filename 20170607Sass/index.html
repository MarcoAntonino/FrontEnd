Rettangolo

Nel CSS non potendo utilizzare logiche, nè mettere variabili. Vi è stata quindi la necessità di creare un linguaggio un file .scss, lo mangia, lo compila e lo rende disponibile.

Ripassa l'autoprefixer.

Useremo codepen.io

Con scss usiamo i @mixin che sostituiscono il clearfix. Il mixin sono delle funzioni. Quindi le richiami all'interno della tua pagina css con il comando include. Ricordiamo che il clearfix serve dentro i contenitori che hanno il float

Oltre ai mixin abbiamo le function. La differenza? Il mixin ritorna una codice di css senza che io debba riscriverlo ogni volta. La function invece mi torna proprio un valore.

Per il ridimensionamento dei font, usiamo sempre Modular scale.

Dal pronpt di comando se scriviamo sass --watch style.scss In questo modo ogni volta che salviamo il .scss il browser refrescia sempre il css compilato. Questo va a richiamare un programmino ruby fatto apposta. Per spegnerlo, premi ctrl + c

Less è la stessa identica cosa ma funziona in Js invece che Ruby, quindi cambia solo il compilatore che ci gira dietro.

>@mixin transition($time){ transition: all $time ease-in-out; } Il transition si mette nell'elemento che sarà interessato da essa. Nel caso volessimo più transition, il primo attributo non sarà all ma "background per esempio"

Altro argomento

Tutte le librerie, i motori, i compilatori sono dati da dei gestori di pacchetti in modo che tutti ce li abbiano.

JS ha npm. In qeusto modo tu ti vai a scaricare il pacchetto che ti serve. Per esempio modularscale-sass.

Non ti mette un file in formato .json. Quando fai npn install, ti va a leggere proprio il json dove ci sono tutti i pacchetti che ti servono e quindi anche gli altri colleghi devono solo fare questo.

Ormai su npm è uno standard e tutte le librerie sono disponibili lì sopra.
